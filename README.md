## Setup Instructions

### 1. Install Dependencies
```bash
npm install
```

### 2. Configure Environment Variables
Copy `.env.example` to `.env` and fill in your project-specific values. The application reads the following keys:

| Variable | Description |
| --- | --- |
| `VITE_FIREBASE_API_KEY` | Firebase web API key |
| `VITE_FIREBASE_AUTH_DOMAIN` | Firebase auth domain (usually `*.firebaseapp.com`) |
| `VITE_FIREBASE_PROJECT_ID` | Firebase project ID |
| `VITE_FIREBASE_STORAGE_BUCKET` | Firebase Storage bucket |
| `VITE_FIREBASE_MESSAGING_SENDER_ID` | Firebase messaging sender ID |
| `VITE_FIREBASE_APP_ID` | Firebase web app ID |
| `VITE_FIREBASE_MEASUREMENT_ID` | (Optional) Analytics measurement ID |
| `VITE_FIREBASE_DATABASE_URL` | (Optional) Realtime Database URL |
| `VITE_AZURE_STORAGE_ACCOUNT_NAME` | Azure storage account used for player assets |
| `VITE_AZURE_STORAGE_SAS_TOKEN` | SAS token generated by `setup-azure.sh` |
| `VITE_GEMINI_API_KEY` | (Optional) Google Gemini key for AI scorecard analysis |
| `VITE_ENABLE_LOGGING` | Set to `true` to force console logging outside development |

### 3. Provision Azure Blob Storage (Media & Stats)
Run the helper script to create the storage account resources and generate a SAS token:

```bash
chmod +x setup-azure.sh
./setup-azure.sh
```

The script will output the `VITE_AZURE_STORAGE_ACCOUNT_NAME` (`islanderscricket`) and a SAS token. Add both values to `.env` and to your hosting provider's environment settings (Netlify or Vercel).

### 4. Deploy Firebase Security Rules
Deploy the Firestore and Storage rules to keep data protected:

```bash
# Install Firebase CLI
npm install -g firebase-tools

# Login to Firebase
firebase login

# Initialize Firebase (select your project)
firebase init

# Deploy rules
firebase deploy --only firestore:rules
firebase deploy --only storage:rules
```

### 5. Run Development Server
```bash
npm run dev -- --host
```

The application will be available at the URL printed in the terminal (default `http://localhost:5173`).

### 6. Run the Test Suite
```bash
npm run test
```

### 7. Build for Production
```bash
npm run build
```

## Deployment

### Netlify (Recommended for Islanders operations team)
1. Push your latest commits to GitHub.
2. In Netlify, select **Add new site → Import an existing project** and connect this repository.
3. When prompted, keep the defaults from `netlify.toml` (`npm run build`, publish directory `dist`).
4. Configure environment variables under **Site settings → Environment variables** (or via CLI):
   - `VITE_FIREBASE_*` values from your Firebase project
   - `VITE_AZURE_STORAGE_ACCOUNT_NAME`
   - `VITE_AZURE_STORAGE_SAS_TOKEN`
   - `VITE_GEMINI_API_KEY` (if AI features are enabled)
5. Trigger the deploy. You can also deploy from your machine with:

   ```bash
   netlify deploy --prod
   ```

Netlify will provide a live URL once the build succeeds.

### Vercel (Alternative)
1. Push your code to GitHub.
2. Import the project into Vercel.
3. Set the same environment variables listed above via `vercel env add` or the dashboard.
4. Deploy with `vercel --prod`.

### Manual Deployment
```bash
npm run build
# Deploy the `dist` folder to your hosting provider
```

## Firebase Setup

### Create Admin User
After setting up Firebase, create an admin user:

```javascript
// Run this in Firebase Console > Authentication
// Or use Firebase Admin SDK
{
  email: "admin@islanders.com",
  password: "your-secure-password",
  displayName: "Admin"
}

// Then in Firestore, create a document in 'users' collection:
{
  uid: "the-auth-uid",
  email: "admin@islanders.com",
  role: "admin",
  displayName: "Admin",
  createdAt: new Date()
}
```

### Create Player Accounts
For each player, create an account and link it to their player profile.
